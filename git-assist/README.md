# Git Assist

Git Assist は、Codex や手作業でアプリ開発・改修を進めるときに、Git/GitHub 操作を安全かつ整理された形で支援する Codex Skill です。個人開発でもチーム開発でも、作業前の確認、コミット前の整理、コミットメッセージ作成、PR 文章作成支援に使います。

## 使い方

`/git-assist` を実行すると、次のモード選択から開始します。

1. 作業前チェック
2. コミット前レビュー
3. コミットメッセージ作成
4. PR文章作成支援

最初から「作業前チェック」「コミット前レビュー」「コミットメッセージ作成」「PR文章作成支援」などを明示した場合は、モード選択を省略して直接その処理を開始できます。

## モード

### 作業前チェック

現在のブランチ、未コミット変更、未追跡ファイル、差分概要、直前コミットを確認します。`main` / `master` / `develop` などの保護ブランチ上にいる場合や、既存変更と新しい作業が混在しそうな場合に注意喚起します。

### コミット前レビュー

コミット前に変更ファイル、差分量、変更内容の要約、意図しない変更の混入、コミット分割の必要性を確認します。必要に応じて軽量な Conventional Commits 風のコミットメッセージ案も提示します。

### コミットメッセージ作成

変更内容を短く要約し、実務的で読みやすいコミットメッセージ案を作成します。デフォルトでは `feat:` / `fix:` / `refactor:` などの英語 prefix と、日本語の簡潔な説明を組み合わせます。

### PR文章作成支援

現在ブランチとベースブランチとの差分を整理し、PR タイトル案と PR 本文案を作成します。実装詳細の羅列よりも、レビューに必要な趣旨、変更箇所、見てほしい観点を優先します。`git push` や `gh pr create` などの GitHub 状態変更操作は実行せず、必要に応じてコマンド案のみ提示します。

## 安全方針

- `git reset --hard`、`git push --force`、`git rebase`、`git stash drop` などの破壊的操作や履歴変更操作は自動実行しません。
- `git commit`、`git push`、`gh pr create`、PR 作成コマンドは、ユーザーが明示的に許可した場合のみ実行します。
- 必要以上に処理範囲を広げず、現在の依頼に必要な Git/GitHub 確認に集中します。
- テスト観点やテスト結果の詳細は、ユーザーが明示的に依頼した場合のみ扱います。

## 出力方針

- コミットメッセージは、そのままコピーできる形式で出力します。
- PR タイトルは、不要な装飾や説明文を付けず、コピーしやすい形式で出力します。
- PR 本文は、GitHub に貼り付け可能な生 Markdown を優先します。
- PR 本文をコードブロックで囲む場合は、ネストしたコードブロックで崩れないよう、多重バッククォートなどを使います。
- 「レビュー方法」は必要な場合のみ記載し、単なる build / lint / test コマンド列挙にしません。
